# Presentation Guide

## How to Present the Project
1. Start with the problem: email overload and spam filtering.
2. Explain the solution: a Streamlit app backed by a trained text classifier.
3. Show the training pipeline: CSV -> TF-IDF -> model search -> saved artifacts.
4. Show the inference path: text or MBOX -> clean -> vectorize -> predict.
5. Close with the latest performance numbers and the current limitations.

## What to Emphasize
- The project is simple to understand and easy to run locally.
- The best model is currently SVM.
- The repo is honest about its constraints: no DB, no API, no CI/CD.
