# Interview Prep

## 15 Technical Interview Questions
1. How does TF-IDF help a spam classifier?
2. Why did you choose classical ML instead of a neural network?
3. Why is SVM performing well on this task?
4. How does `GridSearchCV` work in this project?
5. Why is weighted F1 more informative than accuracy here?
6. How does the inference pipeline ensure it uses the same vocabulary as training?
7. What are the tradeoffs of using pickle for saved models?
8. Why is the current artifact path design fragile?
9. How would you validate the CSV schema before training?
10. How would you make the confidence score more trustworthy?
11. What failure modes exist when parsing MBOX files?
12. How would you test the pipeline without a UI?
13. What would you log in production?
14. How would you make retraining safer?
15. How would you expose this system as an API if needed?

## System Design Questions Demonstrated
- How would you separate training and inference so they can scale independently?
- How would you manage model versions and rollback?
- How would you harden a file-based ML application for production use?
- How would you add observability to a small Streamlit ML tool?

## Behavioral Questions
- Tell me about a time you had to work with an imbalanced dataset.
- Tell me about a time you had to choose between simplicity and extensibility.
- Tell me about a time you had to debug a model artifact mismatch.
- Tell me about a time you improved a workflow by making it more reproducible.
- Tell me about a time you handled a broken dependency or environment mismatch.
