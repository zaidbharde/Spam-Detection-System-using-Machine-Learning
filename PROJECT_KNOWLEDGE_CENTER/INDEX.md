## Document Dependency Map
- [PROJECT_SUMMARY.md](PROJECT_SUMMARY.md) is the top-level factual overview.
- [EXECUTIVE_SUMMARY.md](EXECUTIVE_SUMMARY.md) compresses the same facts for fast reading.
- [PROJECT_STRUCTURE_ANALYSIS.md](PROJECT_STRUCTURE_ANALYSIS.md) and [FILE_ANALYSIS.md](FILE_ANALYSIS.md) feed most downstream docs.
- [ARCHITECTURE.md](ARCHITECTURE.md), [ALGORITHMS.md](ALGORITHMS.md), and [ML_ANALYSIS.md](ML_ANALYSIS.md) explain how the source actually works.
- [SECURITY_REPORT.md](SECURITY_REPORT.md), [CODE_QUALITY_REPORT.md](CODE_QUALITY_REPORT.md), and [RECOMMENDED_STRUCTURE.md](RECOMMENDED_STRUCTURE.md) are the review and remediation layer.
- [DEPLOYMENT_GUIDE.md](DEPLOYMENT_GUIDE.md), [CONFIGURATION_GUIDE.md](CONFIGURATION_GUIDE.md), and [TROUBLESHOOTING.md](TROUBLESHOOTING.md) support operation.
- [presentation/](presentation/) documents are built from the summary, architecture, ML, and risk sections.

## Reader Paths
1. New developer: summary -> structure -> file analysis -> configuration -> testing.
2. Interviewer: executive summary -> architecture -> ML analysis -> security -> presentation prep.
3. Investor: executive summary -> pitch scripts -> performance -> future improvements.
4. Auditor: security -> code quality -> recommended structure -> deployment and troubleshooting.
